How to Recover Data from a Dead Android Phone

21 August 2024 | Posts

Pearl Ramson

Do you have a dead Android phone with precious memories - photos, videos, music - that you thought were lost forever? If so, here’s some good news: recovering data from a seemingly “dead” Android phone is indeed possible.

Many assume that if a phone is dead or unresponsive, data recovery is out of reach. But with the right tools and the steps I will show you here, you can regain access to your files.

A “dead” Android phone, in this context, means any phone that’s unresponsive or inaccessible due to issues like boot loop, forgotten password, bricked device, or hardware malfunction. While these issues may prevent normal access, they don’t necessarily mean your data is gone for good.

If you've already tried multiple software solutions with no success, don’t worry. This tutorial walks you through a proven, 100%-free method to recover data from a dead Android phone with an intact memory. So, let’s get started!

Note:

This method works as long as the phone’s memory hasn’t been wiped through flashing or factory resetting. Stay tuned for updates as we continue exploring ways to recover data from factory-reset or flashed phones. You should know we are committed to satisfying the curiosity of every reader here.

Tools Needed

A personal computer

Internet access

USB cable

Step-by-Step Tutorial

Step 1: Download SP Flash Tool

First, download the latest SP Flash Tool from spflashtools.com in zip format (version 5.2316 seems to be the latest as of 2024). Extract the zip folder to any location on your pc.

========================================================

The SP Flash Tool is primarily used to flash firmware on devices with MediaTek chips. It can also be used to access phone memory with MediaTek chips even without a battery inserted in them. Here, we’ll use it to access the phone’s memory.

Step 2: Download (from this link) and Install MTK VCOM Preloader USB Drivers

This driver allows SP Flash Tool to communicate with your phone even without a battery. Installing this driver is somewhat technical, so follow the installation instructions (from this link) carefully.

Without this driver, SP Flash Tool won’t detect or read the phone.

Step 3: Find and Download Your Phone’s Stock Firmware

Next, download the stock firmware for your specific phone model from your manufacturer’s website or a trusted site.

From the firmware, we have to get the phone’s scatter file. Scatter file usually is a text document that contains essential information about the phone’s memory layout. SP Flash Tool requires this file to work effectively.

Step 4: Load the Scatter File in SP Flash Tool

Launch SP Flash Tool by opening the `flash_tool.exe` file in the folder you extracted earlier - one good thing is the program is plug and play.

By default, you’ll be in the “Download” tab. Click the “Choose” button next to the “Scatter-loading File” line to load your scatter file.

Once the scatter file downloads, you should see something like the picture below.

Ensure you have downloaded the correct scatter file, please!

Step 5: Run a Memory Test

Switch to the “memory Test” tab. Here, you need to run a memory test to get the size of the memory intended for recovery.

Check all boxes under the “Select Memory Test Method” or just click the Select All button below the boxes. Then, click the start button at the right hand side to begin the test.

The test should take some few minutes. Once the test completes, note down the EMMC_PART_USER’s figure shown after the result. The figure indicates the size of the phone memory we want to backup. We will need it later after this process is completed.

At the front of the EMMC_PART_USER, there is Size and then a figure like this: 0x00000001d2800000(7464MB). This shows the connected phone has a 8GB memory.

Step 6: Set Up Readback in SP Flash Tool

Switch over to the “Readback” tab. This is where we need the EMMC_PART_USER’s figure from the memory test.

Click the “Add” button to create a new readback file.

Once completed, double click on the newly created readback file to save the file in a location on your computer.

Upon saving the file, a new page will appear as shown below.

On this page, we will configure the region and length of memory we want to readback. Click the dropdown button in front of “Region” and select EMMC_USER. Leave the start address as it is but change the “Length” address to the EMMC_PART_USER’s figure we got under the “Memory Test” tab.

Now, we are set to readback the memory of the phone. By now, the “Read Back” button is activated. Click on it and wait for the process to complete within some minutes depending on the size of the phone’s memory.

====================================

Step 7: Mount the Readback File with OSFMount

Once step 6 is completed, locate the saved readback file on your PC. You can’t read the file yet. In fact, it is completely meaningless because it is in Hex format unless we have a means to read it.

This is where OSFMount comes in. OSFMount allows us to mount image files as a virtual drive on Windows. With it, we will be able to mount our readback file as a virtual disk so that we can save its content in a meaningful format.

You can download it online from this link.

Open OSFMount after the installation is completed. Click on “Mount new” at the bottom left of the program.

The mounting process has four stages. In the first stage, you have to select the disk image file and the type you want to mount.

Select Disk image file and move on to navigate to the location you saved the readback file earlier on and select it. Click next to move on to stage 2.

In this stage, chose “Mount entire image as virtual disks” and click next.

Quite surprising, OSFMount usually jumps from stage 2 to 4, so you automatically jump to step 4 from step 2 on clicking next.

In stage 4, check the “Mount as removable media” and leave all other options in their default selection. Then, click on mount to mount the virtual disk on Windows.

OSFMount will mount the virtual disk by assigning it letters from “E” to “Z”.

However, it is very common that the files may not be accessible immediately because a Windows prompt usually appears asking to format the drive before it opens.

This means the original readback file is not a valid system file.

We won’t be able to access the virtual drives on the computer with this situation. But, there is a solution.

We need to find a good data recovery software to access the virtual drive. You can try any data recovery software you know but Power Data Recovery software seems to be the best for this task. You can check it here.

Step 8: Recover Files with Power Data Recovery Software

Download Power Data Recovery software from the link in step 7. It doesn’t need installation because it is a plug and play program. Another good news is that it is fantastic and free so far it is for home use.

Open the program and click on “Lost Partition Recovery”.

We have to assume we want to recover data from the lost virtual drives.

The program will scan for all attached devices and, luckily, our mounted virtual drives are included, too.

Once the attached devices appear, you have to click on the virtual drives one after the other.

Watch out for drives with heavy size because they usually have the highest number of files. In fact, the drive with the highest size usually contains phones’ data.

Before we move on, we have to set the extent and type of files we want to recover.

Click on the “Advanced Setting” button to go to the page.

For best recovery, check all boxes and click ok.

Now, highlight any of the virtual disk and click on the “Full Scan” button. The scan will commence and should take some moments depending on the disk size.

On completion, you should have something like the image below:

Then, firstly highlight the partition, then click on “Show Files” at the top left hand side of the program.

Here we have our files in various format. You can navigate through them as you desire.

To recover files in any of the folder, right-click on the folder and select “Recover” from the options available.

Select a location to save the files and begin the process.
